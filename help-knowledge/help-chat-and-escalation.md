# Help chat, limits, and reporting bugs

## Opening help

Use the **Help** button (lower-right). This opens the in-app help assistant backed by the JobDock knowledge base.

## Scope (JobDock only)

This assistant answers **only** questions about **using JobDock** (screens, workflows, troubleshooting in the product, subscriptions within JobDock, bug reports via Send report). It is **not** a general-purpose AI: homework, unrelated coding projects, trivia, drafting unrelated content, and other topics are intentionally out of scope; use an external assistant for those.

## What it can do

- Explain how features work using official knowledge.  
- Give **sequential troubleshooting** steps grounded in product facts.  
- Redirect you to **Send report to engineering** when something is a bug, unclear, or unsupported.

## Daily limit

There is a **per-user daily turn limit** (configurable via environment; default order of tens of messages per day). If you hit the limit, the message tells you to try tomorrow or use **Send report**.

## Send report to engineering

Use this for:

- Definite or suspected **bugs**  
- **Data** that looks wrong after refresh  
- **Blocking** issues where knowledge base has no path  

The report emails the product team with **tenant**, **user**, optional **summary**, optional **client route** (which page you were on), and a **transcript** of the recent chat. One report per session is deduplicated if you click twice.

To make reports actionable, include: what you clicked, expected vs actual, time (with timezone), document IDs (quote/invoice/job) if relevant.

## Settings → Help vs floating chat

**Settings → Help** tab describes the same floating chat and tutorial; it is not a different bot.

## Contact support email

Email from **Settings → Help** is a normal mailto for human support — use for account issues or when email is easier than chat.
